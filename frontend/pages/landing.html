The Architecture

How It Works Under The Hood

1. The PaySim Dataset

Our model is trained on a synthetic dataset representing mobile money transactions. By analyzing millions of legitimate and fraudulent cash-outs and transfers, the system learns the subtle differences between normal behavior and organized theft.

2. Graph Feature Extraction

Before making a prediction, the backend constructs a NetworkX directed graph. It calculates critical graph-theoretic metrics for every account involved in a transaction:

  • PageRank: Identifies central "hub" accounts often used for laundering.
  • In/Out Degree: Detects anomalies in receiving vs. sending volumes.
  • Clustering Coefficient: Spots tightly knit groups of accounts coordinating illicit transfers.

3. The Ensemble Model

Because financial fraud is extremely imbalanced (fraud occurs in less than 0.1% of transactions), standard models fail. We utilize a Balanced Random Forest Classifier that is highly sensitive to minority classes, combined with SMOTE (Synthetic Minority Over-sampling Technique).

4. Tech Stack

The system operates entirely via a REST API. The backend is powered by Python (Flask), Scikit-Learn, and NetworkX. The frontend uses a proprietary vanilla JavaScript component framework with Chart.js for analytics and HTML Canvas for live topology rendering.
